# مسح الذاكرة المؤقتة وإعادة تعيين التطبيق

إذا واجهت "خطأ غير متوقع" أو بطيء في الواجهة، اتبع هذه الخطوات لإعادة التطبيق إلى حالة نظيفة.

1) تحديث الصفحة (بسيط):
   - اضغط Ctrl+F5 (Windows) أو Cmd+Shift+R (Mac) لإعادة تحميل العنصر وتجاوز الكاش.

2) مسح حافظات المتصفح (Cache) وملفات التخزين المحلي (LocalStorage / SessionStorage):
   - افتح Developer Tools (F12).
   - اذهب إلى Application > Local Storage > (حذف مفاتيح التطبيق) أو نفّذ التالي في Console:
     ```javascript
     // تحذير: هذا سيمسح بيانات التطبيق المحلي
     localStorage.clear();
     sessionStorage.clear();
     indexedDB.databases().then(dbs => dbs.forEach(db => indexedDB.deleteDatabase(db.name)));
     ```
   - اغلق المتصفح ثم أعد تشغيله.

3) إعادة تثبيت تطبيق الويب (PWA):
   - إذا كنت تستخدم PWA أو تثبيت على الهاتف، أزل التطبيق (Uninstall) ثم قم بتثبيته مجدداً من المتصفح.

4) إذا كان الجهاز بطيئاً بشكل عام (هاتف أو حاسوب):
   - تأكد أن النسخة الحديثة من المتصفح تعمل.
   - أغلق علامات التبويب غير المستخدمة.
   - قم بإيقاف الخدمات أو الإضافات التي تستهلك الذاكرة (extensions).

5) إذا كانت مشكلة السرعة على صفحة "الادمن" بسبب استدعاء بيانات كثيرة:
   - افتح DevTools > Network, وراجع Requests الضخمة أو الطبقات (throttling).
   - نموذج سريع: التأكد من أن صفحة الادمن لا تقوم بتحميل جميع المتاجر قبل وجود فلتر أو بحث.

6) نصيحة للمطورين (تحسين الحمل والسرعة):
   - Lazy-load المكونات الثقيلة (Admin tables, charts) باستخدام React.lazy
   - تفعيل شرط `prefers-reduced-motion` لتقليل الرسوم المتحركة على الأجهزة الضعيفة
   - استخدم pagination أو virtualized lists للجدوال الكبيرة (react-window / react-virtualized)
   - تجنب توليد عناصر متحركة كثيرة في DOM (مثل جزيئات في الخلفية)

إذا كنت تريد، أستطيع: 
- إضافة lazy-loading إضافي للمكونات الكبيرة في AdminPanel.
- تنفيذ pagination للسجلات الضخمة داخل لوحة الادمن لتسريع الحمل.
